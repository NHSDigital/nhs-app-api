Code Review Checklist
=====================

Refer to https://confluence.service.nhs.uk/display/NO/Peer+review+checklist for more details.

- [ ] Does the code actually achieve the task at hand? Have you seen it? Does it look correct?
- [ ] Do you understand what the code is doing?
- [ ] Is the code consistent with existing code?
- [ ] Is the code defensive/at the right level of abstraction? Is it a copy/paste that requires refactoring?
- [ ] Is the code fully tested?
- [ ] Would you be happy if the code was made open source? Are there any secrets in it?
- [ ] Does the commit message start with the Jira ticket number, and is it descriptive enough?
- [ ] Have the commits been squashed?
- [ ] Are there any warnings or static analysis problems?
- [ ] Are there any performance or security implications?
- [ ] Is the amount of logging appropriate? Can you work out what's going wrong based on logs alone?
- [ ] Does it look how it's supposed to look?
- [ ] Has the WAVE toolbar been run to test accessibility?
